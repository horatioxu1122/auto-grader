# Grading Rubric: HW3 Recursive Fibonacci

## Correct use of recursion (30 points)
- Full credit: Pure recursive implementation with correct base cases
- Partial credit (15-25): Recursion used but with minor issues (e.g., missing base case for n=0)
- Minimal credit (5-10): Hybrid approach using both loops and recursion
- No credit: Purely iterative solution

## Code clarity (20 points)
- Full credit: Clean, readable code with meaningful variable names
- Partial credit: Code works but is hard to follow
- No credit: Incomprehensible or heavily obfuscated

## Output correctness (30 points)
- Full credit: Correct Fibonacci sequence printed for first 10 numbers
- Partial credit: Mostly correct with minor errors
- No credit: Incorrect output or no output

## Explanation quality (20 points)
- Full credit: Clear docstring/comments explaining the recursive approach
- Partial credit: Some explanation but incomplete
- No credit: No explanation at all
